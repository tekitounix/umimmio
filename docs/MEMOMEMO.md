## 2. アーキテクチャの役割分担（暫定）

- Controller: ターゲット非依存の領域を担当
- App: Config で要求を表明
- OS: ケイパビリティベースで要求に応答
- App は最低限のヘッドレス動作を実装
	- ヘッドレス部分 + 互換機能が、あらゆるターゲットで動作することを目標

## 4. 組み込み環境のI/O設計方針

- 組み込み環境のみを想定
	- 動的なポート変更は不要
- USBデバイス/拡張ボード接続でI/Oが変化しても、メモリ資源は有限
	- 静的確保した範囲内で動作する設計にする
- 拡張デバイス接続時は span で渡すポート数を変化させる必要
