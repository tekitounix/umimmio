# STM32 Cortex-M7 DMA と DCache 実用ワークアラウンド

## 概要

Cortex-M7 の DCache は write-back 方式であり、DMA はキャッシュを参照しない。
そのため DMA と CPU 間のメモリ整合性はハードウェアでは保証されず、ソフトウェア設計による対策が必須である。
本ドキュメントは STM32H7 F7 系マイコンで実運用に耐えるワークアラウンドを簡潔にまとめたものである。

---

## 基本原則

* DMA と CPU が同一キャッシュラインを同時に触らない
* キャッシュ操作は最小化し一箇所に集約する
* 人が直接 SCB_CleanDCache 系 API を呼ばない

---

## 推奨ワークアラウンド

### 1. DMA 専用メモリ領域の分離

* DMA 用バッファ専用セクションを定義する
* MPU で Non-cacheable または Write-through に設定する
* 通常ヒープと完全に分離する

効果

* キャッシュ操作が不要
* バグ耐性が最も高い

---

### 2. DMA トランザクション API の導入

DMA 前後の処理を API に閉じ込める。

```
dma_begin_tx(buf, size, dir);
dma_end_tx(buf, size, dir);
```

内部規約

* CPU から DMA へ渡す前に clean
* DMA から CPU へ渡した後に invalidate

効果

* 人的ミスの排除
* ドライバ責務の明確化

---

### 3. キャッシュライン単位の制約

* アラインメントは 32 bytes
* サイズは 32 bytes の倍数

効果

* 部分 clean invalidate による副作用を回避

---

### 4. DMA 受信後に clean しない

* DMA が書いたデータに対して clean を行わない
* 受信後は invalidate のみを使用する

効果

* DMA 結果の破壊を防止

---

### 5. ダブルバッファ構成

* DMA 用バッファと CPU 用バッファを分離
* memcpy 境界のみでキャッシュ操作を行う

効果

* 高頻度 DMA 転送で安定
* USB Audio I2S 等に有効

---

### 6. Peripheral 主導設計

* CPU は制御のみ担当
* データ転送は DMA 主導で完結
* CPU はリングバッファの head tail のみ操作

効果

* キャッシュ衝突点を最小化

---

## 実行効率への影響

DMA と DCache 対策は実行効率に影響を与える。設計判断の指針を以下に示す。

### キャッシュ操作のコスト

* SCB_CleanDCache 系はキャッシュライン単位で実行される
* 数十〜数百サイクルかかり、割り込み不可区間が発生する
* 高頻度呼び出しはリアルタイム性を破壊する

原則としてサンプル単位や小転送ごとに呼ばない。

---

### Non-cacheable 領域の性能特性

* CPU アクセスは遅くなる
* DMA 転送性能は変わらない
* 制御用途やストリーム用途では問題になりにくい

DMA 専用バッファでは許容されることが多い。

---

### ダブルバッファの効率

* memcpy はキャッシュ上で高速に実行される
* キャッシュ操作回数を大幅に削減できる
* 総合的な CPU 使用率は下がることが多い

リアルタイムオーディオ用途で最もバランスが良い。

---

### Peripheral 主導設計の効率

* CPU と DMA の競合がほぼ消える
* キャッシュヒット率が向上
* レイテンシとジッタが安定する

設計段階で最優先すべき構成。

---

### DCache 無効化の実行効率

* CPU 処理は確実に遅くなる
* 400MHz クラスでは多くの用途で実用範囲
* バグ調査と安定性は飛躍的に向上

製品安定性最優先の場合は合理的な選択。

---

## 条件付き対策

### DCache を無効化

* 帯域に余裕がある場合
* 製品安定性を最優先する場合

STM32H7 400MHz クラスでは現実的な選択肢である。

---

## 非推奨事項

* DMA 前後で SCB API を直接手書きする
* 非アライン DMA バッファの使用
* DMA 受信後の clean 呼び出し
* DMA バッファを通常ヒープに混在させる

---

## 結論

* 自動的な解決策は存在しない
* 正解はコードではなくメモリ設計
* DMA は専用領域に閉じ込める

Cortex-M7 はキャッシュを理解して使う前提のマイコンである。
