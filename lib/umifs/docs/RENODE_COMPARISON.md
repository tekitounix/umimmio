# Renode比較レポート: C++23ポート vs 参照C実装

対象: STM32F407VG (Cortex-M4), Renode エミュレーション
ツールチェーン: clang-arm 19.1.5, -Os
日付: 2026-02-02

## テスト結果

| | C++23ポート | 参照C実装 |
|---|---|---|
| littlefs テスト | 15/15 PASS | 15/15 PASS |
| FATfs テスト | 13/13 PASS | 13/13 PASS |
| **合計** | **28/28 PASS** | **28/28 PASS** |

## Flashサイズ

| | C++23ポート | 参照C | 差分 |
|---|---|---|---|
| Flash | 78,412 B | 69,768 B | +8,644 (+12.4%) |
| RAM (data+bss) | 123,704 B | 123,720 B | -16 (同等) |

サイズ増加の主因: C++ namespace/テンプレート、inline メンバ関数のコード重複。
RAM はほぼ同等（テスト用 RAM ブロックデバイスが支配的）。

## DWT_CYCCNT ベンチマーク (サイクル数)

### littlefs

| 操作 | C++23ポート | 参照C | 比率 |
|---|---|---|---|
| format+mount | 89,520 | 88,896 | 1.007x |
| write 1KB | 218,420 | 217,486 | 1.004x |
| read 1KB | 40,889 | 40,574 | 1.008x |
| mkdir+stat x5 | 911,810 | 903,503 | 1.009x |

littlefs は全操作で 1% 未満の差。実質同等性能。

### FATfs

| 操作 | C++23ポート | 参照C | 比率 |
|---|---|---|---|
| write 4KB | 57,225 | 91,688 | 0.624x |
| read 4KB | 25,723 | 57,337 | 0.449x |
| mkdir+stat x3 | 189,162 | 162,691 | 1.163x |

FATfs の read/write は C++23 ポートが大幅に高速。参照 C 実装はグローバル関数経由の
diskio 呼び出し（間接呼び出し）がオーバーヘッドとなっている。mkdir+stat は 16% 遅いが、
メタデータ操作は安全性改善（循環チェーン検出等）のコストを含む。

## 安全性改善（C++23ポートのみ）

C++23 ポートには以下の安全性改善が含まれており、上記の性能で実現されている:

### littlefs
- L1: ASSERT → エラーリターン（7箇所）— プロダクションビルドでのスタック保護
- L2: lfs_npw2 の UB ガード
- L4: LFS_THREADSAFE 時の lock/unlock 静的検証

### FATfs
- F1: 2nd FAT 書き込みエラーチェック
- F2: FSInfo 書き込みエラーチェック
- F3/F6: 循環クラスタチェーン検出（全走査箇所）
- F4: truncate の条件付き objsize 更新
- F5: rename 中間 sync（クロスリンク窓縮小）
