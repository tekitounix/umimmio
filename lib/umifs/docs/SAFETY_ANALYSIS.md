# ファイルシステム安全性分析: 割り込み安全性・電源遮断耐性

## 概要

littlefs と FATfs の C++23 ポートにおける割り込み安全性と電源遮断耐性の分析。

## littlefs

### 電源遮断耐性: ✓ 設計レベルで保証

| メカニズム | 詳細 |
|-----------|------|
| **メタデータペア** | 各ディレクトリは pair[0]/pair[1] の2ブロックで冗長化。書き込みは常に非アクティブ側に行い、完了後にスワップ |
| **CRC-32検証** | 全コミットにCRC-32。読み込み時にバリデーション。不整合検出で自動フォールバック |
| **アトミックコミット** | CCRC（コミットCRC）とFCRC（フォワードCRC）による2段階検証 |
| **COW（Copy-on-Write）** | データブロックはin-place更新せず、新規ブロックに書き込み後にポインタ更新 |
| **gstate復旧** | 不完全なmove/delete操作はgstateに記録され、次回マウント時に復旧 |

### 電源遮断シナリオ

1. **メタデータ書き込み中**: pair[0]は未変更。pair[1]の不完全書き込みはCRC失敗で無視→pair[0]にフォールバック
2. **データ書き込み中**: 新規ブロックへのCOW。旧データは健全。メタデータ未更新なら旧ポインタが有効
3. **リロケーション中**: gstateが不完全移動を追跡。次回マウントでリトライ

### 割り込み安全性

- **明示的なロック/割り込み禁止コード: なし**
- **読み込みは安全**: CRC検証により、書き込み中の不整合データを検出・排除
- **ISRからのFS操作は非推奨**: ブロックデバイスI/Oがブロッキング
- **推奨**: FS操作はタスクコンテキストでのみ実行。ISRからはキューを経由

### 組込みでの注意点

```
ISRからの直接的なFS呼び出し → ✗ 禁止（ブロッキングI/O）
タスクからの排他的アクセス  → ✓ 推奨
電源遮断からの復旧          → ✓ mount()呼び出しのみで自動復旧
```

## FATfs

### 電源遮断耐性: ✗ 保証なし

| メカニズム | 詳細 |
|-----------|------|
| **FAT二重化** | プライマリFAT + セカンダリFAT。しかし2回の独立した書き込みで非アトミック |
| **セクタバッファ** | 512Bのウィンドウバッファ。wflagでダーティ管理。sync時に書き出し |
| **検証なし** | 書き込み後の読み戻しCRC検証なし |
| **ジャーナルなし** | トランザクション機構なし |

### 電源遮断シナリオ

1. **FAT更新中**: プライマリFAT書き込み後、セカンダリFAT書き込み前に遮断→FAT不整合。fsck必要
2. **ファイルデータ書き込み中**: マルチセクタ書き込みの途中で遮断→部分書き込み。ディレクトリエントリ未更新でサイズ不整合
3. **ディレクトリエントリ更新中**: FATチェーンは存在するがディレクトリが参照しない→リーク。fsck必要

### 割り込み安全性

- **FS_LOCK = 0**: ファイルロック無効
- **FS_REENTRANT = 0**: リエントラント保護無効
- **ISRからの操作**: 完全に安全ではない。共有バッファ(fs->win)が破壊される可能性
- **並行アクセス**: 複数タスクからの同時操作でFAT/ディレクトリ破壊

### 組込みでの推奨対策

```
ISRからの直接的なFS呼び出し → ✗ 禁止
タスクからの排他的アクセス  → ✓ 必須（mutexで保護）
電源遮断からの復旧          → △ fsckツール必要 or sync()頻繁呼び出し
sync()のタイミング          → 書き込み完了ごとに呼び出し推奨
```

## 比較表

| 観点 | littlefs | FATfs |
|------|----------|-------|
| アトミック書き込み | CRC検証付きコミット | セクタ単位（非アトミック） |
| メタデータ冗長化 | デュアルペア | FAT二重化（非アトミック） |
| 電源遮断復旧 | mount()で自動 | fsckツール必要 |
| データ検証 | 全読み込みでCRC | なし |
| ウェアレベリング | ブロックリロケーション | なし |
| 割り込み安全性 | 読み込み安全（CRC） | 安全でない |
| リエントラント | 不要（CRC復旧） | 無効（FS_REENTRANT=0） |

## 結論

- **信頼性が最重要な組込み用途**: littlefs を推奨
- **FAT互換性が必要な場合**: FATfs使用時はmutexによる排他制御とsync()の頻繁呼び出しが必須
- **両FSとも**: ISRからの直接操作は禁止。タスクコンテキストでのみ使用
