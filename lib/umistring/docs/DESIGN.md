# umistring 設計ドキュメント

## 目的

組み込み環境で安全に使用できる軽量文字列処理ライブラリを提供する。

## 設計思想

### リアルタイム安全性

- **ヒープ割り当て禁止**: すべての処理はスタックまたはコンパイル時定数のみ
- **constexpr 対応**: 可能な限りコンパイル時計算
- **例外不使用**: エラーは `Result<T>` またはデフォルト値で返す

### 軽量性

- ヘッダオンリー
- 依存関係最小化
- 使用しない機能はリンク時に除外（ODR利用）

## 主要コンポーネント

| コンポーネント | 説明 | ファイル |
|--------------|------|---------|
| string_view | 文字列ビュー | `string.hh` |
| convert | 型変換 | `convert.hh` |
| split | 文字列分割 | `split.hh` |
| trim | 空白処理 | `trim.hh` |

## API設計

### string_view

std::string_view の代替。組み込み環境で std::string_view が使えない場合に使用。

```cpp
constexpr string_view sv{"Hello"};
sv.size();           // 5
sv[0];               // 'H'
sv.substr(0, 3);     // "Hel"
```

### to_string

整数を文字列に変換。バッファは呼び出し側で用意。

```cpp
char buf[16];
auto result = to_string(42, buf);
// result == "42"
```

### split

文字列を分割。結果は静的配列に格納（組み込み向け）。

```cpp
auto parts = split("a,b,c", ',');
// parts[0] == "a"
// parts[1] == "b"
// parts[2] == "c"
```

## 制約

- 最大文字列長: 256バイト（組み込み向け制限）
- 分割結果最大数: 16個
- すべての関数は noexcept

## 依存関係

- C++23 標準ライブラリのみ
- 外部ライブラリなし

## パフォーマンス

- 文字列長 N に対して O(N)
- メモリ割り当てゼロ
- キャッシュフレンドリー（連続メモリアクセス）
